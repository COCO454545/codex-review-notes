# 03_新项目创建SOP

> 本 SOP 用于规范每一个新项目的创建、命名、目录结构、资料归档和交付方式。Codex 执行新项目时必须先参考本文档。

## 一、什么时候必须创建项目

出现以下任一情况，都应创建项目文件夹：

1. 用户要求做新的工具、应用、视频、文档、自动化、调研、实验或交付物。
2. 当前任务不属于任何已有 `Project_编号_项目名称` 文件夹。
3. 任务会产生多个文件、素材、脚本、截图、输出或过程记录。
4. 用户要求长期复用、持续迭代或后续维护。
5. 任务需要保存上下文、复盘、版本或验收记录。

以下情况可以不创建项目：

1. 简单问答、翻译、解释。
2. 一次性命令查询。
3. 不产生文件、不需要后续维护的小任务。
4. Workspace 级别的 Codex 使用复盘、全局偏好沉淀、`AGENTS.md`/SOP/hooks/skills 规则更新。
5. GitHub 同步规则、全局文档整理、跨电脑复用说明。此类内容直接更新根目录全局 md。

判断原则：

- 产生“业务交付物”或“项目资产”时，创建 `Project_*`。
- 产生“全局规则、偏好、SOP、复盘方法、同步规范”时，不创建项目，直接沉淀到根目录全局文档。

## 二、项目命名规范

项目文件夹统一命名：

```text
Project_编号_项目名称
```

命名要求：

1. `Project` 固定使用英文首字母大写。
2. 编号使用两位数字，从 `01` 开始递增。
3. `Project_00_全局工作台` 为全局保留目录，不参与普通项目编号。
4. 项目名称简短、明确、可搜索。
5. 中文、英文、数字、下划线均可使用。
6. 不使用空格、斜杠、冒号、问号、星号等特殊符号。

## 三、标准目录

```text
Project_编号_项目名称
├─ 00_project_brief.md
├─ 01_requirements
├─ 02_references
├─ 03_assets
├─ 04_src
├─ 05_outputs
├─ 06_tests
├─ 07_logs
├─ 08_review
└─ 99_temp
```

## 四、创建流程

### Step 1：确认项目编号

在 `D:\Workspace` 中查看已有项目：

```powershell
Get-ChildItem -Directory -Filter "Project_*" |
  Where-Object { $_.Name -match '^Project_\d{2}_' } |
  Sort-Object Name
```

取最大普通项目编号加一。`Project_00_全局工作台` 不计入普通项目编号。

### Step 2：创建项目文件夹和目录

推荐命令：

```powershell
$name = "Project_编号_项目名称"
New-Item -ItemType Directory -Force -Path $name | Out-Null
"01_requirements","02_references","03_assets","04_src","05_outputs","06_tests","07_logs","08_review","99_temp" | ForEach-Object {
  New-Item -ItemType Directory -Force -Path (Join-Path $name $_) | Out-Null
}
New-Item -ItemType File -Force -Path (Join-Path $name "00_project_brief.md") | Out-Null
```

### Step 3：填写项目简报

`00_project_brief.md` 至少包含：

```text
# Project_编号_项目名称

## 项目目标

## 用户需求

## 当前状态

## 验收标准

## 目录说明

## 关键文件

## 下一步
```

### Step 4：更新全局项目索引

在 `02_全局项目索引.md` 追加项目记录，至少填写：

- 编号
- 项目名称
- 文件夹
- 状态
- 项目目标
- 下一步
- 更新时间

### Step 5：开始执行

文件按类型放入对应目录：

- 需求、聊天摘要、任务说明：`01_requirements`
- 参考图、参考网页、竞品资料：`02_references`
- 图片、视频、音频、字体、Logo：`03_assets`
- 代码、脚本、配置、工程文件：`04_src`
- 最终导出、交付版本、成品：`05_outputs`
- 测试报告、截图、验证记录：`06_tests`
- 执行日志、报错记录、命令记录：`07_logs`
- 复盘、反馈、优化建议：`08_review`
- 临时中转文件：`99_temp`

## 五、旧版目录映射

如果从旧 Workspace 或外部项目迁移资料，可以按下面方式归入新结构：

| 旧目录 | 新目录 |
| --- | --- |
| `00_项目说明.md` | `00_project_brief.md` |
| `01_需求与目标` | `01_requirements` |
| `02_素材与资料` | `02_references` 或 `03_assets` |
| `03_过程文件` | `04_src`、`06_tests`、`07_logs` 或 `99_temp` |
| `04_交付成果` | `05_outputs` |
| `05_复盘与日志` | `08_review` 或 `07_logs` |

## 六、外部文件处理规则

1. 用户提供的截图、图片、压缩包、文档等外部文件，如果会被项目使用，应复制或移动到项目目录。
2. 参考资料放入 `02_references`，原始素材放入 `03_assets`。
3. 如果文件体积很大且用户要求保留原位置，应在项目简报中记录原路径和用途。
4. 不依赖临时目录中的文件作为长期项目资产。

## 七、项目完成标准

项目完成前检查：

- [ ] 交付物已放入 `05_outputs`。
- [ ] 源文件或脚本已放入 `04_src`。
- [ ] 关键素材已放入 `03_assets` 或 `02_references`。
- [ ] 测试或验证记录已放入 `06_tests`。
- [ ] 重要过程日志已放入 `07_logs`。
- [ ] 复盘内容已放入 `08_review`。
- [ ] 可复用经验已同步到 `01_全局复利踩坑日志.md`。
- [ ] 项目状态已更新到 `02_全局项目索引.md`。
- [ ] `99_temp` 已清理或归档。

## 八、禁止事项

1. 禁止把项目文件直接放在 Workspace 根目录。
2. 禁止把项目资料长期放到桌面、下载目录、临时目录或其它无关位置。
3. 禁止多个项目混用一个文件夹。
4. 禁止只输出最终文件而不保留关键过程资料。
5. 禁止把临时文件长期留在 `99_temp`。
6. 禁止把全局规则复盘误建成普通项目文件夹。
