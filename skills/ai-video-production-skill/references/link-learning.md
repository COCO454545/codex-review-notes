# Link Learning

Use this when the user shares another creator's video link, downloaded clip, screenshots, transcript, or says "learn from this video".

## Goal

Extract reusable production patterns, not copy the source video. Convert good references into the user's own style, workflow, templates, and future prompts.

## Capture First

Record these in `reference-learning.md` inside the active project:

- Source link and platform.
- Date analyzed.
- Available evidence: link only, screenshots, downloaded clip, transcript, audio, comments, title, cover.
- What cannot be verified from the available evidence.

If only a link is available and the platform content cannot be fetched directly, ask for screenshots, a downloaded clip, transcript, or screen recording only when needed. Continue with visible metadata when enough context exists.

## Analyze

Use this grid:

| Area | What to extract | Reusable output |
|---|---|---|
| Hook | First 1-3 seconds, promise, curiosity gap | Opening line pattern |
| Structure | Scene order, information rhythm, reveal timing | Storyboard pattern |
| Visual style | Layout, density, colors, typography, motion language | Style note and preview idea |
| Oral delivery | Tone, speed, sentence length, pauses, emotional contour | Narration rules |
| Materials | Screenshots, demos, charts, b-roll, UI captures | Asset checklist |
| Editing | Cuts, zooms, captions, sound hits, transitions | Motion/cut rules |
| Platform packaging | Title, cover, tags, comment hook | Publish package notes |

When the source is a social screenshot or poster, analyze both the image and the visible caption:

| Area | What to inspect | Reusable output |
|---|---|---|
| Cover formula | title, number, color emphasis, promise | First-frame composition |
| Card system | rank labels, icons, tags, preview blocks, density | Reusable layout skeleton |
| Decision aid | priority tiers, recommendations, "who should use it" | Bottom summary strip |
| Caption voice | conversational sentence pattern, warning, curiosity line | Spoken hook and publish caption |
| Engagement clues | likes, favorites, comments, shares if visible | Why this packaging likely works |

For GitHub/project-analysis topics, add a second research pass:

| Area | What to inspect | Why |
|---|---|---|
| README positioning | headline, first paragraph, screenshots, install path | Finds the project's real promise |
| Demo/proof | GIFs, videos, examples, screenshots, benchmark claims | Prevents generic commentary |
| Docs/examples | quickstart, integrations, templates | Shows adoption friction |
| Releases/issues/discussions | recent changes, pain points, user questions | Explains why stars may be accelerating |
| Ecosystem fit | Codex, Claude Code, Cursor, MCP, RAG, short-video stack | Connects project to current AI workflows |

## Translate Into Production

After analysis:

1. Add 3-7 reusable rules to the project's `video-spec.md` or `DESIGN.md`.
2. Add one small style preview only if it will be reused.
3. Add a "do not copy" note when a pattern is close to the source.
4. Promote durable lessons to the skill only if they apply beyond one project.

## Quality Bar

The final adapted video should feel inspired by the reference's strengths, but it must use the user's own topic, structure, visuals, narration, and assets.
