# 02_全局项目索引

> 本文档是 `D:\Workspace` 的项目看板。它只记录普通 `Project_*` 项目的基本状态，帮助在项目层快速知道当前有哪些项目在做、暂停、完成或归档。

## 一、状态定义

| 状态 | 含义 |
| --- | --- |
| `规划中` | 已创建项目，但目标、范围或资料还在整理 |
| `进行中` | 当前正在执行或近期会继续推进 |
| `暂停` | 暂时不推进，但未来可能恢复 |
| `已完成` | 已交付，主要成果已整理 |
| `已归档` | 已完成整理、复盘和归档，短期不再改动 |

## 三、更新规则

1. 新建普通项目后，必须追加一行。
2. 项目状态变化时，更新 `状态`、`下一步` 和 `更新时间`。
3. 项目完成后，确认关键输出、测试和复盘已整理，再标记为 `已完成`。
4. 项目短期不再改动时，可标记为 `已归档`。
5. 不删除历史项目行；废弃项目也应保留记录。
6. 全局规则、偏好、SOP、skills 调整这类沉淀层工作，不进入本索引，不新增 `Project_*`。

## 四、注意事项

- 本文档只管理普通项目层，不管理 Workspace 根目录全局 md 和 `skills/`。
- 如果某个项目中的经验已经升格为跨项目通用方法，应转写到：
  - `01_全局复利踩坑日志.md`
  - `06_工作区提示词.md`
  - `D:\Workspace\skills\`
